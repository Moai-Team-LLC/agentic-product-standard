# Architecture Decision Records

Why this repo is shaped the way it is. Each ADR captures one decision: the
context, the choice, and its consequences — so a future contributor understands
the *why*, not just the *what*.

## Format

Use `0000-template.md`. Number sequentially (`NNNN-kebab-title.md`). Status is
one of **Proposed · Accepted · Superseded by ADR-XXXX · Deprecated**. Keep each
ADR to one decision and one page.
